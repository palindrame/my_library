                                                              BRANCH PREDICTOR
###############################################################################################################################################
The branch predictor is also a dedicated hardware component:

Location: Physically located in the CPU's front-end, between the instruction fetch unit and the decoder
Physical proximity: Sits very close to the instruction cache (I-cache) but is a separate component from it
Integration: Not part of the MMU or memory controller

Implementation: Consists of:

      1. Pattern History Tables (PHTs): SRAM arrays storing branch history
      2. Branch Target Buffer (BTB): A specialized cache storing branch target addresses
      3. Return Address Stack (RAS): A small hardware stack for function return prediction
      4. Global/local history registers: Shift registers tracking branch patterns
      5. Predictor logic: Circuitry implementing prediction algorithms



The branch predictor works closely with the instruction fetch unit to determine the next instruction address, 
before the current branch is resolved. Its outputs feed directly into the program counter logic and instruction fetch mechanisms.
In modern CPU die photos, we can typically see the branch predictor as a distinct block near the L1 instruction cache
and fetch unit in the front-end of the pipeline, while reservation stations appear as regular structures near their
respective execution units throughout the execution core.
