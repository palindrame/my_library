====================================================
  TECHNICAL NOTE: ModRM and SIB Bytes in x86/x64
====================================================

INTRODUCTION:
-------------
In x86/x86-64 assembly, memory addressing often uses a combination
of base registers, index registers, scale factors, and displacement.
To encode such instructions compactly in machine code, the CPU uses:

- ModRM byte (1 byte)
- SIB byte (1 byte, optional)
- Displacement (0, 1, or 4 bytes, optional)

These allow instructions to express memory access modes like:
    [base + index * scale + displacement]

--------------------------
 1. ModRM BYTE STRUCTURE
--------------------------
ModRM is a required byte in most instructions involving memory or register operands.

Format (8 bits):
    +----+-----+------+
    |Mod | Reg | R/M  |
    +----+-----+------+
     2b   3b    3b

Fields:
- MOD (2 bits): addressing mode (memory vs register)
- REG (3 bits): register operand (destination or source)
- R/M (3 bits): register or memory operand (base or direct)

MOD FIELD VALUES:
    00 -> memory with no displacement (unless R/M = 101)
    01 -> memory with 8-bit displacement
    10 -> memory with 32-bit displacement
    11 -> register-direct (no memory)

EXAMPLES:
Instruction:    mov eax, [ebx]      
Encoded as:     Mod = 00, Reg = 000 (eax), R/M = 011 (ebx)

Instruction:    mov eax, [ebx + 4]  
Encoded as:     Mod = 01, Reg = 000, R/M = 011 (ebx), 8-bit disp = 0x04

If R/M = 100 → SIB byte follows.

--------------------------
 2. SIB BYTE STRUCTURE
--------------------------
SIB is used when the ModRM R/M field = 100 (special case for memory addressing).

Format (8 bits):
    +--------+--------+--------+
    | Scale  | Index  | Base   |
    +--------+--------+--------+
      2 bits   3 bits   3 bits

Fields:
- SCALE: multiplier for the index register (00=1, 01=2, 10=4, 11=8)
- INDEX: register used as an index (or 100 = none)
- BASE:  register used as the base

EXAMPLE:
Instruction:    mov eax, [ebx + ecx*4 + 8]

Breakdown:
- BASE  = EBX
- INDEX = ECX
- SCALE = 4
- DISP  = 8

Encoding:
- ModRM: Mod = 10 (disp32), Reg = 000 (eax), R/M = 100 → SIB follows
- SIB: Scale = 10 (4), Index = 001 (ecx), Base = 011 (ebx)
- Displacement: 08 00 00 00

-------------------------------
 3. REGISTER CODES (3 bits)
-------------------------------
Common 3-bit codes for registers in x86 (32-bit):

| Code | Register |
|------|----------|
| 000  | EAX      |
| 001  | ECX      |
| 010  | EDX      |
| 011  | EBX      |
| 100  | ESP      |
| 101  | EBP      |
| 110  | ESI      |
| 111  | EDI      |

Note:
- R/M = 100 triggers SIB
- R/M = 101 with MOD=00 → special meaning (uses disp32 as absolute address)

---------------------------------
 4. COMMON ADDRESSING FORMS
---------------------------------

(1) [EBX]
    → ModRM: 00 000 011
    → No SIB or displacement

(2) [EBX + 4]
    → ModRM: 01 000 011
    → Disp8: 0x04

(3) [EBX + ECX * 4]
    → ModRM: 00 000 100
    → SIB:   10 001 011
    → No displacement

(4) [EBX + ECX * 4 + 8]
    → ModRM: 10 000 100
    → SIB:   10 001 011
    → Disp32: 0x08 0x00 0x00 0x00

(5) [0x12345678]
    → ModRM: 00 000 101
    → No SIB
    → Disp32: 0x78 0x56 0x34 0x12

---------------------------------------
 5. WHY SIB IS IMPORTANT
---------------------------------------
- Enables complex addressing: base + index * scale + offset
- Great for accessing arrays, structures, or matrix elements
- Keeps instruction size compact using only 1 or 2 extra bytes

------------------------
 6. SUMMARY
------------------------

- ModRM is used in most instructions with register/memory operands.
- SIB is used when memory involves both base and scaled index.
- They allow flexible, efficient encoding of memory addresses.
- Displacement can be 0, 1, or 4 bytes depending on the mode.
- Instruction decoders parse these to compute effective address.
