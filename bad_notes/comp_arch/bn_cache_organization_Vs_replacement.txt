==============================================================
      RELATIONSHIP BETWEEN CACHE ORGANIZATION AND 
              REPLACEMENT ALGORITHMS
==============================================================

Terminology:
------------
* Cache Mapping / Organization: How memory blocks are mapped to cache lines.
  - Direct-Mapped: Each memory block maps to exactly one cache line.
  - Fully Associative: A memory block can be placed in any cache line.
  - N-Way Set Associative: Cache is divided into sets; each set holds N blocks.

* Replacement Algorithm: Decides which block to evict when inserting a new block
  into a set or the whole cache (if full).

--------------------------------------------------------------
1. CACHE MAPPING ORGANIZATIONS
--------------------------------------------------------------

(1) DIRECT-MAPPED CACHE
------------------------
* Description: 1 block per set (N = 1).
* Mapping: `Index = (Block Address) mod (Number of Sets)`
* Replacement Policy: **None**
  - Since only one block can exist in a set, if a new block maps to the same set,
    it *must* replace the existing one.
  - No choice => no need for replacement policy.

(2) FULLY ASSOCIATIVE CACHE
-----------------------------
* Description: 1 set containing all cache lines (N = total number of lines).
* Mapping: Block can go **anywhere**.
* Replacement Policy: **Required**.
  - Since any line can be replaced, we must choose one.
  - Policies: LRU, LFU, FIFO, Random, etc. (see below).

(3) N-WAY SET ASSOCIATIVE CACHE
-------------------------------
* Description: Cache is divided into multiple sets, each with N lines (N > 1).
* Mapping: `Index = (Block Address) mod (Number of Sets)`
* Replacement Policy: **Required per-set**.
  - Within each set, when all N blocks are filled, a replacement decision must be made.
  - Same algorithms (LRU, LFU, etc.) apply at **set level**, not entire cache.

--------------------------------------------------------------
2. DEPENDENCY: CACHE ORGANIZATION vs REPLACEMENT ALGORITHMS
--------------------------------------------------------------

❗ KEY POINT:
Replacement **algorithms and cache mapping** are **distinct concepts**, but:

✅ **Certain algorithms are only practical or meaningful in certain organizations.**

--------------------------------------------------------------------------------------------
| Replacement Algorithm  | Direct-Mapped   | Fully Associative   | N-Way Set Associative   |
|------------------------|-----------------|---------------------|-------------------------|
| None (always replace)  | ✅ Required     | 🚫 Not valid        | 🚫 Not valid            |
| Random                 | 🚫 Not needed   | ✅ Yes              | ✅ Yes (per-set)        |
| LRU                    | 🚫 Not needed   | ✅ Yes              | ✅ Yes (per-set)        |
| Tree-PLRU              | 🚫 Not needed   | 🚫 Too expensive    | ✅ Yes (per-set)        |
| LFU                    | 🚫 Not needed   | ✅ Yes              | ✅ Yes (per-set)        |
| FIFO                   | 🚫 Not needed   | ✅ Yes              | ✅ Yes (per-set)        |
| MRU                    | 🚫 Not needed   | ✅ Yes              | ✅ Yes (per-set)        |
| ARC / LIRS             | 🚫 Not needed   | ✅ Yes              | ✅ Sometimes (complex)  |

Explanation:
------------
- Direct-Mapped: No choice => no algorithm needed.
- Fully Associative: Must track entire cache to choose victim block.
- N-Way Set Associative: Algorithm is applied **within each set** of size N.

--------------------------------------------------------------
3. EXAMPLES
--------------------------------------------------------------

Example A: 4-Way Set Associative Cache
--------------------------------------
- Each set has 4 lines.
- LRU can be applied per-set using 4-way age tracking or 3-bit tree-PLRU.
- Eviction occurs only when all 4 lines in a set are filled.

Example B: Fully Associative Cache in TLBs (Translation Lookaside Buffers)
--------------------------------------------------------------------------
- Any page entry can go anywhere.
- Replacement algorithms like **NRU**, **Clock**, **LFU** commonly used.

Example C: Direct-Mapped Instruction Cache
------------------------------------------
- Each instruction block maps to a single line.
- If a new instruction maps to the same line, the old one is evicted, no choice.

--------------------------------------------------------------
4. SUMMARY TABLE
--------------------------------------------------------------

| Cache Mapping       | Needs Replacement Policy? | Replacement Scope   |
|---------------------|----------------------------|----------------------|
| Direct-Mapped       | ❌ No                      | Not applicable       |
| Fully Associative   | ✅ Yes                     | Whole cache          |
| N-Way Set Assoc.    | ✅ Yes                     | Within each set      |

--------------------------------------------------------------
5. CONCLUSION
--------------------------------------------------------------

- Cache mapping **determines the degrees of freedom** for block placement.
- Replacement algorithms **resolve conflicts** when there’s more than one place to choose from.
- So:
  🔹 **Mapping constrains**,  
  🔹 **Replacement selects** within those constraints.

They are **orthogonal**, but **interdependent in implementation**.
