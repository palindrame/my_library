
In operating systems and processor architecture, a "trap" refers to a specific type of synchronous transfer of control,
from a program to the operating system kernel or a designated handler. 
It is typically caused by the execution of an instruction or a condition that requires special handling.

----------------------------------------------------------------------------------------------------------------------------------
Terminology Consensus
---------------------
The terminology around traps, exceptions, and interrupts can vary across different architectures, 
operating systems, and textbooks. Here's a breakdown of the most commonly agreed-upon definitions:

        Trap:

            A synchronous transfer of control to the kernel
            Usually intentional and expected
            Often caused by a specific instruction (trap instruction, system call)
            Returns control to the next instruction after handling


        Exception:

            A synchronous transfer of control to the kernel
            Usually unexpected or error-related
            Caused by conditions like division by zero, page faults, or illegal instructions
            May not return to the same execution point (may abort the program)


        Interrupt:

            An asynchronous transfer of control to the kernel
            Caused by external events independent of the currently executing instruction
            Examples: I/O completion, timer expiration, hardware signals
            Returns to wherever the program was interrupted (with some exceptions)

----------------------------------------------------------------------------------------------------------------------------------
Key Distinguishing Features
---------------------------

        Timing:

            Traps/Exceptions: Synchronous (occur at specific points in program execution)
            Interrupts: Asynchronous (can occur at any time)


        Origin:

            Traps: Usually explicitly invoked by programs
            Exceptions: Result from error conditions or specific instruction execution
            Interrupts: Come from external devices or timer hardware


        Return Behavior:

            Traps: Return to next instruction
            Exceptions: May return to faulting instruction or abort
            Interrupts: Return to interrupted instruction
