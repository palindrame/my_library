

⚖️  System Calls vs. Library Functions: Performance Considerations


  🛠 System Calls

      Definition: Direct requests to the kernel to perform privileged operations (I/O, memory, etc.).

      Performance: Slower than pure user-mode functions because they involve a context switch to kernel mode.

      Use Case: Necessary when only the OS can do the operation (file I/O, changing CPU affinity, etc.).

  📚 Library Functions

      Definition: Functions provided by standard libraries (like glibc) running in user space.

      Performance: Faster when the work can be done entirely in user space.

      NOTE: Some library functions (like malloc, printf, etc.) may internally invoke system calls.


  🚀 Which is More Performant?

      Use library functions whenever possible for speed.

      Use system calls only when necessary (e.g., file I/O, affinity setting).

      For fine-grained performance benchmarking (like RDTSC), ensure you minimize context switches and 
      pin the process to a CPU core to avoid migration-induced noise.
